# AI Assets Marketplace

## Как запустить проект

### 1. Установка зависимостей
Рекомендуется использовать `pnpm`.

```bash
pnpm install
```

Если `pnpm` не установлен, можно использовать `npm`:

```bash
npm install
```


### 2. Запуск в режиме разработки

```bash
pnpm dev
```

или

```bash
npm run dev
```

Откройте в браузере:

```
http://localhost:3000
```


### 3. Сборка для продакшена

```bash
pnpm build
```

или

```bash
npm run build
```


### 4. Запуск собранного приложения

```bash
pnpm start
```

или

```bash
npm run start
```


### 5. Проверка кода

```bash
pnpm exec eslint 'app/**/*.{ts,tsx}' 'components/**/*.{ts,tsx}' 'lib/**/*.{ts,tsx}' --no-ignore --max-warnings=0
```


## Основные страницы

- `/` — главная страница магазина
- `/checkout` — оформление заказа
- `/admin` — админ-панель
- `/admin/orders` — страница заказов для админа


## Админ-панель

Админ-панель скрыта от обычных покупателей и продавцов: в публичной навигации сайта нет ссылки на неё.

Чтобы попасть в админ-панель, нужно открыть напрямую URL `/admin`.

Для входа используйте демо-пароль:

- `admin123`

После входа админ может:

- редактировать цену ассетов
- включать/выключать акцию `On sale`
- переключать флаг `Featured`
- просматривать созданные mock-заказы
- переходить на страницу заказов `/admin/orders`


## Как работает админ-авторизация

- Флаг входа сохраняется в `localStorage`
- Если пользователь не вошёл, `/admin` показывает страницу входа
- `/admin/orders` тоже требует входа, иначе перенаправляет на страницу авторизации


## Локальные данные

Проект использует mock-данные и сохраняет изменения в `localStorage`:

- изменения ассетов (`price`, `isOnSale`, `isFeatured`)
- mock-заказы из checkout
